JavaScript Basics

JavaScript is a cross-platform, object-oriented scripting language. JavaScript is extremely popular for a variety of reasons. It is a small and lightweight language allowing maximum flexibility for developers to take it in a buch of different directions. JavaScript lives inside a host envirnment (a web browser or node server), it can be connected to objects of these envirnments to provide programic control over them.

  • Variable Declaration JavaScript variables are containers for storing data values - imagine a cup you fill with coffee, the cup holds the coffee, a variable holds a value. All JavaScript must be identified with unique names. These unique names are called identifiers var x

  • Variable Assignment Assignment operators assign values to JavaScript varibles - our cup can now have coffee poured in it, giving our variable a value to hold.The = assigment operator assigns a value to a variable. var x = 10;

  • Data Types Data Types are an important concept; to be able to operate on varibles you need to know data type. There are six data types that are JavaScript primitives. Boolean true or false; null - null aka nothing; Number- 42, or 3.14159; String - "Coding Dojo Rocks"; Array - ['Coding',2,'Dojo']; and Object - {first_name:'Jane',last_name:'Doe'}

  • Conditionals When you write code, you want to perform different actions for different decisions - hitting differnt code blocks base on values or conditions that have been met. You can use conditional statements in your code to accomplish this. There are the following conditional statements: if a specified condition is true, do this code in our code block; else if to specify a new condition to test, if the first conditon is false; else we execute this block of code;

  • Loops There are many different kinds of loops in every programming language, but they all essentially do the same thing; they will repeat an action some number of times. Imagine you have to run a mile, well you run around the track four times and then stop. Thats a loop!

  • Function Functions are an encapsulation of code block. When we call our function this will run that code block. Thik of it as a list of instructions. As an example imagine we are putting together a desk from Ikea, we ope up the instruction manual and get started, first we screw the legs to the table top; next we place the table right way up. Done! We finished our instructions. Sadly there are a ton more tables to do so lets call our function over and over and over again.
